# 碳排放资产管理系统项目说明文档

## 一、项目概述

### 1.1 项目背景

随着全球气候变化问题日益严重，碳排放管理成为企业和政府关注的重点。本项目旨在开发一个碳排放资产管理系统，帮助用户实现碳排放数据的记录、分析和管理，提高碳排放管理的效率和准确性。

### 1.2 项目目标

- 实现用户管理功能，包括注册、登录和个人信息管理

- 实现碳排放资产的录入、查询和统计分析

- 提供 AI 助手服务，辅助用户进行碳排放管理决策

- 采用前后端分离架构，提高系统的可维护性和扩展性

### 3.2 分层设计

**后端分层**

1. Controller 层：处理 HTTP 请求

2. Service 层：业务逻辑

3. Mapper 层：数据访问

4. Entity 层：数据库实体

5. DTO 层：数据传输对象

6. Common 层：工具类

**前端分层**

1. View 层：页面组件

2. Layout 层：布局组件

3. Router 层：路由配置

4. Utils 层：工具类

5. Components 层：通用组件

## 五、系统流程

### 5.1 用户登录流程

```Plain Text

1. 用户访问登录页面
2. 输入用户名和密码
3. 前端发送登录请求
4. 后端验证
5. 返回用户信息
6. 前端存储信息并跳转主页
```

### 5.2 碳排放资产录入流程

```Plain Text

1. 用户进入资产录入页面
2. 填写资产信息
3. 前端验证
4. 发送请求
5. 后端存储
6. 返回成功提示
```

## 七、总结

本项目基于前后端分离架构，实现了一个功能完整的碳排放资产管理系统。系统采用现代化技术栈，具有良好的可维护性和扩展性。系统功能包括用户管理、碳排放资产管理和 AI 助手服务，能够满足用户的碳排放管理需求。

通过本项目的开发，团队掌握了前后端分离架构的开发流程，熟悉了 Spring Boot、Vue 3 等技术栈的使用，提高了团队协作和项目管理能力。同时，本项目也为碳排放管理提供了一个有效的工具，具有一定的实际应用价值。
